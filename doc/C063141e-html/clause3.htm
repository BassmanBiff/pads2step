ISO 10303-21:2016(E)

3 Terms, definitions, and abbreviated terms

3.1 Term and definitions

For the purposes of this document, the following terms and definitions apply.

3.1.1 Term defined in ISO/IEC 8859-1

  • byte.

3.1.2 Terms defined in ISO/IEC 10646

  • basic multilingual plane;
  • character;
  • code point;
  • encoding scheme;
  • graphic character;
  • octet.

3.1.3 Terms defined in ISO 10303-1

  • application protocol;
  • exchange structure.

3.1.4 Terms defined in ISO 10303-11

  • complex entity instance;
  • data type;
  • EXPRESS constant;
  • entity;
  • partial complex entity instance;
  • simple entity instance;
  • token.

3.1.5 Terms defined in ISO/IEC 16262

  • ECMAScript.

3.1.6 Terms defined in ISO/IEC 21320-1

  • ZIP Archive.

3.1.7 Other terms and definitions

3.1.7.1
Uniform Resource Identifier
URI

compact string of characters for identifying an abstract or physical resource

[Source: IETF RFC 2396: Uniform Resource Identifiers (URI): Generic Syntax]

3.1.7.2
URI Fragment Identifier

additional reference information to be interpreted after a URI retrieval action has been successfully completed

[Source: Fragment identifier in IETF RFC 2396]

3.1.7.3
URI Query component

string of information to be interpreted by a resource

[Source: Query component in IETF RFC 2396]

3.1.7.4
Universally Unique Identifier
UUID

identifier that is unique across both space and time

[Source: IETF RFC 4122: A Universally Unique IDentifier (UUID) URN Namespace]

3.1.7.5
Base64 encoding

character encoding of the CMS

[Source: IETF RFC 4648: The Base16, Base32 and Base64 Data Encodings]

3.1.7.6
Cryptographic Message Syntax
CMS

specification for the form and content of a digital signature

[Source: IETF RFC 5652: Cryptographic Message Syntax (CMS)]

3.1.7.7
Message digest

hash value computed on the content and included in the signature data

[Source: IETF RFC 5652 Section 5]

3.1.7.8
anchor

identifier that can be addressed in a reference

3.1.7.9
basic alphabet

set of UCS characters corresponding to code points U+0020 to U+007E and U+0080 to U+10FFFF

3.1.7.10
clear text encoding

encoding of information, using a sequence of codes for characters in the basic alphabet

3.1.7.11
control directive

sequence of characters in the basic alphabet

3.1.7.12
keyword

special sequence of characters identifying an entity or a defined type in the exchange structure

3.1.7.13
reference

address of an anchor in an exchange structure, or in another type of resource that can be converted to an exchange structure

3.1.7.14
resource

physical or abstract exchange structure addressed by a URI

3.1.7.15
section

collection of data of the same functional category of information

3.1.7.16
sequential file

file that can only be accessed in a sequential manner

3.1.7.17
signature

cryptographically encoded authentication of content

3.1.7.18
tag

additional information about an anchor that is outside the scope of the schema

3.1.7.19
token separator

sequence of one or more characters in the basic alphabet that separate any two tokens
